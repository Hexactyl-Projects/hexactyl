Configuration Guide

Configure Hexactyl for your specific needs

Environment Variables

Hexactyl is configured through environment variables in the .env file. This section covers all available configuration options.

Application Settings

Core application configuration options:

  • APP_NAME - Application name displayed in the panel (default: Hexactyl)
  • APP_ENV - Environment type: production, development, or testing
  • APP_KEY - Encryption key for secure data (generate with: openssl rand -base64 32)
  • APP_URL - Full URL to your panel (e.g., https://panel.yourdomain.com)
  • APP_DEBUG - Enable debug mode: true or false (never use true in production)
  • APP_MAINTENANCE_MODE - Enable maintenance mode: true or false

Complete Environment Example

# Application Settings
APP_NAME=Hexactyl
APP_ENV=production
APP_KEY=base64:your_generated_key_here
APP_URL=https://panel.yourdomain.com
APP_DEBUG=false
APP_MAINTENANCE_MODE=false

# Locale Settings
APP_LOCALE=en
APP_FALLBACK_LOCALE=en
APP_TIMEZONE=UTC

# Encryption
CIPHER=AES-256-CBC
⚠️ Security Note: Never commit your .env file to version control. Always keep your APP_KEY secret and use strong, unique passwords for all services.

Mail Configuration

Configure email sending for user notifications, password resets, and system alerts.

SMTP Configuration

Configure SMTP settings for email delivery:

# Mail Settings
MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="${APP_NAME}"

Supported Mail Drivers

  • smtp: Standard SMTP server (most common)
  • sendmail: PHP's mail() function
  • mailgun: Mailgun API integration
  • ses: Amazon SES integration
  • postmark: Postmark API integration
  • log: Log emails to file (development only)

Testing Email Configuration

Test your email configuration by sending a test email:

# Test email configuration
docker exec hexactyl-panel php artisan tinker

# In tinker, run:
Mail::raw('Test email', function($message) {
  $message->to('test@example.com')->subject('Test');
});
💡 Gmail Users: If using Gmail, you'll need to generate an App Password. Go to Google Account → Security → 2-Step Verification → App passwords.

S3 Backup Storage

Configure S3-compatible storage for automatic server backups:

AWS S3 Configuration

# S3 Backup Settings
BACKUP_DRIVER=s3
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=hexactyl-backups
AWS_ENDPOINT=null
AWS_USE_PATH_STYLE_ENDPOINT=false

MinIO Configuration

For self-hosted S3-compatible storage (MinIO):

# MinIO S3 Configuration
BACKUP_DRIVER=s3
AWS_ACCESS_KEY_ID=your_minio_access_key
AWS_SECRET_ACCESS_KEY=your_minio_secret_key
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=hexactyl-backups
AWS_ENDPOINT=https://minio.yourdomain.com
AWS_USE_PATH_STYLE_ENDPOINT=true

Backup Settings

  • BACKUP_DRIVER - Storage driver: s3, s3cdn, or local
  • BACKUP_PATH - Local backup path (default: /var/lib/hexactyl/backups)
  • BACKUP_RETENTION - Days to keep backups (default: 30)

Local Storage Configuration

# Local Backup Settings
BACKUP_DRIVER=local
BACKUP_PATH=/var/lib/hexactyl/backups
BACKUP_RETENTION=30
⚠️ Important: Always ensure your backup storage has sufficient space and consider implementing a backup rotation strategy to manage storage costs.

Docker Daemon Configuration

Configure the Docker daemon for optimal performance with Hexactyl:

Docker Daemon Settings

Edit /etc/docker/daemon.json to optimize Docker for game servers:

{
  "storage-driver": "overlay2",
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "default-ulimits": {
    "nofile": {
      "Name": "nofile",
      "Hard": 65535,
      "Soft": 65535
    }
  },
  "iptables": true,
  "ip-forward": true
}

Docker Compose Optimization

Optimize the Docker Compose configuration for production:

version: '3.8'

services:
  panel:
    restart: unless-stopped
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '0.5'
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

Network Configuration

Configure Docker networking for game server traffic:

# Create a custom network
docker network create --driver bridge --subnet=172.20.0.0/16 hexactyl-network

# Add to docker-compose.yml
networks:
  default:
    external: true
    name: hexactyl-network

Security Settings

Configure security settings to protect your Hexactyl installation:

Authentication Settings

  • AUTHENTICATION_METHOD - Login method: username, email, or both
  • REQUIRE_EMAIL_VERIFICATION - Require email verification: true or false
  • SESSION_LIFETIME - Session timeout in minutes (default: 120)
  • MAX_LOGIN_ATTEMPTS - Max failed login attempts before lockout (default: 5)
  • LOCKOUT_TIME - Account lockout duration in minutes (default: 15)

Two-Factor Authentication

Enable two-factor authentication for enhanced security:

# 2FA Settings
TWO_FACTOR_ENABLED=true
TWO_FACTOR_GRACE_PERIOD=15
TWO_FACTOR_REMEMBER_BROWSER=true

Rate Limiting

Configure rate limiting to prevent abuse:

# Rate Limiting
RATE_LIMIT_ENABLED=true
RATE_LIMIT_MAX_ATTEMPTS=5
RATE_LIMIT_DECAY_MINUTES=1

Firewall Configuration

Configure UFW firewall rules for security:

# Allow SSH
ufw allow 22/tcp

# Allow HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# Allow game server ports (as needed)
ufw allow 25565/tcp # Minecraft

# Enable firewall
ufw enable
⚠️ Critical Security: Always keep your system updated, use strong passwords, enable 2FA, and regularly review access logs for suspicious activity.

Advanced Configuration

Additional configuration options for advanced users:

Queue Configuration

Configure background job processing:

# Queue Settings
QUEUE_CONNECTION=redis
QUEUE_DRIVER=redis
REDIS_HOST=cache
REDIS_PORT=6379
REDIS_PASSWORD=null
REDIS_DATABASE=1

Logging Configuration

Configure application logging:

# Logging Settings
LOG_CHANNEL=daily
LOG_LEVEL=info
LOG_PATH=/var/log/hexactyl

File Storage

Configure file storage for uploads:

# File Storage
FILESYSTEM_DISK=local
STORAGE_ROOT=/var/lib/hexactyl/storage
MAX_UPLOAD_SIZE=102400 # in kilobytes
💡 Configuration Tips: After making changes to the .env file, restart the containers with: docker compose restart
