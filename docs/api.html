API Documentation

Integrate with Hexactyl using our comprehensive REST API

Overview

The Hexactyl API allows you to interact with your panel programmatically. Use it to automate server management, integrate with external tools, or build custom applications.

Base URL

All API endpoints are relative to your panel URL:

https://your-panel-url.com/api

API Versions

  • Client API: /api/client - User-facing endpoints
  • Application API: /api/application - Admin-level endpoints
  • Remote API: /api/remote - Daemon communication

Response Format

All responses are returned in JSON format:

{
  "data": {
    // Response data here
  },
  "meta": {
    "pagination": {
      "total": 100,
      "per_page": 50,
      "current_page": 1
    }
  }
}

Authentication

Authenticate your API requests using API keys:

Generating API Keys

  1. Log in to your Hexactyl panel
  2. Go to Account Settings → API Keys
  3. Click Create New Key
  4. Give your key a descriptive name
  5. Set an expiration date (optional but recommended)
  6. Select the permissions (scopes) for this key
  7. Click Create and copy your key
⚠️ Security Warning: API keys are only shown once. Copy it immediately and store it securely. Never commit API keys to version control.

Using API Keys

Include your API key in the Authorization header:

curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://your-panel-url.com/api/client/servers

API Key Scopes

Available permission scopes:

  • client - Access client API endpoints
  • application - Access application API endpoints
  • remote - Access remote API endpoints
  • servers.* - Full server management access
  • users.* - Full user management access

Example Request

# List all servers
curl -H "Authorization: Bearer abc123..." \
  https://panel.example.com/api/client/servers

# Response
{
  "data": [
    {
      "id": 1,
      "name": "My Minecraft Server",
      "status": "running"
    }
  ]
}

Client API Endpoints

Endpoints for regular users to manage their servers:

Servers

GET /api/client/servers

List all servers the authenticated user has access to.

GET /api/client/servers/{id}

Get details about a specific server.

POST /api/client/servers/{id}/command

Send a command to a server console.

POST /api/client/servers/{id}/power

Control server power state (start, stop, restart, kill).

Files

GET /api/client/servers/{id}/files

List files in a server's directory.

GET /api/client/servers/{id}/files/contents

Read the contents of a file.

POST /api/client/servers/{id}/files/upload

Upload files to a server (multipart/form-data).

PUT /api/client/servers/{id}/files/write

Write content to a file.

DELETE /api/client/servers/{id}/files/delete

Delete files or directories.

Databases

GET /api/client/servers/{id}/databases

List all databases for a server.

POST /api/client/servers/{id}/databases

Create a new database for a server.

DELETE /api/client/servers/{id}/databases/{databaseId}

Delete a database.

Application API Endpoints

Admin-level endpoints for panel management:

Users

GET /api/application/users

List all users on the panel.

POST /api/application/users

Create a new user.

PUT /api/application/users/{id}

Update user details.

DELETE /api/application/users/{id}

Delete a user.

Nodes

GET /api/application/nodes

List all nodes in the panel.

POST /api/application/nodes

Create a new node.

GET /api/application/nodes/{id}

Get node details including system statistics.

Servers

GET /api/application/servers

List all servers on the panel.

POST /api/application/servers

Create a new server.

PUT /api/application/servers/{id}

Update server details.

DELETE /api/application/servers/{id}

Terminate and delete a server.

Templates

GET /api/application/templates

List all server templates.

POST /api/application/templates

Create a new server template.

DELETE /api/application/templates/{id}

Delete a server template.

Rate Limiting

The API implements rate limiting to prevent abuse and ensure fair usage:

Rate Limit Headers

Every API response includes rate limit information:

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 59
X-RateLimit-Reset: 1640995200

Limits by Endpoint

  • Client API: 60 requests per minute
  • Application API: 30 requests per minute
  • Authentication: 5 requests per minute

Handling Rate Limits

When you exceed the rate limit, you'll receive a 429 response:

{
  "error": "Too Many Requests",
  "message": "Rate limit exceeded. Please wait before making more requests."
}
💡 Best Practices: Implement exponential backoff when retrying requests. Cache responses where possible to reduce API calls.

Error Responses

Common error codes and their meanings:

  • 400: Bad Request - Invalid parameters
  • 401: Unauthorized - Invalid or missing API key
  • 403: Forbidden - Insufficient permissions
  • 404: Not Found - Resource doesn't exist
  • 422: Unprocessable Entity - Validation error
  • 429: Too Many Requests - Rate limit exceeded
  • 500: Server Error - Internal server error

Error Response Format

{
  "error": "Validation Error",
  "message": "The given data was invalid.",
  "errors": {
    "name": ["The name field is required."]
  }
}

Code Examples

Example API usage in various programming languages:

cURL

# List servers
curl -X GET \
  https://panel.example.com/api/client/servers \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

# Send command to server
curl -X POST \
  https://panel.example.com/api/client/servers/1/command \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"command": "list"}'

Python

import requests

API_KEY = "your_api_key_here"
BASE_URL = "https://panel.example.com/api"

headers = {
  "Authorization": f"Bearer {API_KEY}",
  "Content-Type": "application/json"
}

# List servers
response = requests.get(f"{BASE_URL}/client/servers", headers=headers)
servers = response.json()["data"]

# Send command
response = requests.post(
  f"{BASE_URL}/client/servers/1/command",
  headers=headers,
  json={"command": "list"}
)

JavaScript (Node.js)

const axios = require('axios');

const API_KEY = 'your_api_key_here';
const BASE_URL = 'https://panel.example.com/api';

const client = axios.create({
  baseURL: BASE_URL,
  headers: {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  }
});

// List servers
const servers = await client.get('/client/servers');

// Send command
await client.post('/client/servers/1/command', {
  command: 'list'
});
