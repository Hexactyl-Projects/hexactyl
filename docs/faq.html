Frequently Asked Questions

Find answers to common questions about Hexactyl

General Questions

What is Hexactyl?

Hexactyl is a modern, open-source game server management panel. It allows you to manage multiple game servers (Minecraft, ARK, Rust, etc.) from a single web interface. Hexactyl provides features like user management, automated backups, file management, database administration, and a RESTful API for automation.

Is Hexactyl free to use?

Yes, Hexactyl is completely free and open-source. You can use it for personal or commercial purposes without any licensing fees. The source code is available on GitHub under the MIT license.

What games does Hexactyl support?

Hexactyl supports a wide range of games including:

  • Minecraft (Java and Bedrock editions)
  • ARK: Survival Evolved
  • Rust
  • Valheim
  • Terraria
  • CS:GO / CS2
  • Garry's Mod
  • And many more through custom eggs

Can I run Hexactyl on a VPS?

Yes, Hexactyl is designed to run on VPS (Virtual Private Servers). It works great on providers like DigitalOcean, Vultr, Hetzner, Linode, and AWS EC2. We recommend at least 2GB RAM for the panel itself, plus additional resources for your game servers.

Installation Questions

What are the system requirements?

Minimum requirements for Hexactyl:

  • OS: Ubuntu 20.04+, Debian 11+, or CentOS 8+
  • RAM: 2GB minimum (4GB recommended)
  • Storage: 10GB free disk space
  • Docker: Docker 20.10+ and Docker Compose 2.0+

For game servers, you'll need additional resources depending on the games you're hosting.

Can I install Hexactyl on my local machine?

While possible for development purposes, Hexactyl is designed to run on a remote server. For production use, we recommend a VPS or dedicated server. Local installations may have networking issues and aren't suitable for hosting game servers that others need to access.

How do I update Hexactyl?

To update Hexactyl to the latest version:

cd /etc/hexactyl
git pull origin main
docker compose pull
docker compose up -d

Always backup your database and configuration before updating.

Do I need to install Docker separately?

Yes, Docker and Docker Compose must be installed before installing Hexactyl. Our installation guide includes instructions for installing Docker on various Linux distributions.

Configuration Questions

How do I enable HTTPS?

To enable HTTPS, you'll need to configure a reverse proxy (Nginx or Apache) with SSL certificates. We recommend using Let's Encrypt for free SSL certificates. See our Installation Guide for detailed instructions.

How do I change the panel URL?

Update the APP_URL variable in your .env file:

APP_URL=https://your-new-url.com

Then restart the containers: docker compose restart

Can I use MySQL instead of MariaDB?

Yes, Hexactyl works with MySQL 8.0, MariaDB 10.5+, and PostgreSQL 13+. MySQL is the default and most tested option.

How do I configure email settings?

Email configuration is done through environment variables in the .env file. See our Configuration Guide for detailed mail setup instructions.

What is the default admin login?

There is no default admin account. During the first installation, you'll be guided through creating your admin account through the web interface. If you lose access, you can reset it via the command line:

docker exec -it hexactyl-panel php artisan p:user:make --admin

Server Management Questions

How do I add a new node?

To add a new node:

  1. Go to Admin Panel → Nodes
  2. Click Create New Node
  3. Enter the node details (name, FQDN, port)
  4. Copy the node token
  5. Install the daemon on the new server using the token

Can I move servers between nodes?

Yes, but it requires manual migration. You'll need to:

  1. Backup the server data
  2. Create a new server on the target node
  3. Restore the backup to the new server
  4. Update DNS if needed
  5. Delete the old server

How do I create backups?

Hexactyl supports automated backups. To set up backups:

  1. Go to your server panel
  2. Click Backups
  3. Click Create Backup for manual backups
  4. Or configure scheduled backups in Schedules

You can store backups locally or on S3-compatible storage.

How do I access server files?

Use the built-in file manager:

  1. Go to your server panel
  2. Click Files
  3. Browse, upload, download, and edit files directly in the browser

You can also use SFTP with the credentials shown in the server panel.

Can I give other users access to my servers?

Yes, Hexactyl has a comprehensive user permission system:

  1. Go to your server panel
  2. Click Users
  3. Click Add User
  4. Enter the user's email and select permissions

You can grant specific permissions like console access, file management, or full control.

Troubleshooting Questions

My server won't start. What should I check?

If your server won't start:

  • Check the server console for error messages
  • Verify the server has enough RAM and disk space
  • Ensure the startup command is correct
  • Check if the game version is available
  • Verify network ports are not blocked
  • Check Docker logs: docker compose logs

I can't access the panel. What's wrong?

If you can't access the panel:

  • Verify the panel containers are running: docker compose ps
  • Check if ports 80/443 are open
  • Verify your DNS is pointing to the server
  • Check Nginx configuration
  • Review container logs: docker compose logs webserver

How do I view logs?

To view logs:

# Panel logs
docker compose logs panel

# Daemon logs
docker compose logs daemon

# All logs
docker compose logs

You can also view logs in the panel under Admin → Diagnostics.

How do I reset my admin password?

To reset your admin password:

docker exec -it hexactyl-panel php artisan p:user:reset-password your-email@example.com

A new password will be displayed. Change it immediately after logging in.

The panel is running slowly. How can I improve performance?

To improve performance:

  • Increase server RAM if possible
  • Enable Redis caching (included in default setup)
  • Optimize MySQL settings
  • Use SSD storage for better I/O
  • Check for resource-intensive processes
  • Consider using a CDN for static assets
