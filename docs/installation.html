Installation Guide

Complete instructions for installing Hexactyl on your server

System Requirements

Before installing Hexactyl, verify your system meets these requirements:

Operating System

  • Ubuntu: 20.04 LTS or newer (recommended)
  • Debian: 11 (Bullseye) or newer
  • CentOS: 8 or newer
  • Rocky Linux: 8 or newer
  • AlmaLinux: 8 or newer

Hardware Requirements

  • CPU: 2 cores minimum, 4+ cores recommended
  • RAM: 2GB minimum, 4GB+ recommended
  • Storage: 10GB minimum, 50GB+ recommended for game servers
  • Network: 10Mbps+ bandwidth

Software Requirements

  • Docker: Version 20.10 or newer
  • Docker Compose: Version 2.0 or newer
  • Git: Latest version from package manager
  • cURL: For downloading installation scripts
⚠️ Root Access Required: Installation requires root or sudo access. All commands in this guide assume you're running as root.

Step-by-Step Installation

Follow these steps to install Hexactyl on a fresh server:

1. Update System Packages

Start by updating your system packages to ensure you have the latest security updates:

# For Ubuntu/Debian
apt update && apt upgrade -y

# For CentOS/Rocky
dnf update -y

2. Install Docker

Docker is required for Hexactyl to function. Install Docker using the official convenience script:

curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# Start and enable Docker
systemctl start docker
systemctl enable docker

3. Install Docker Compose

Install Docker Compose plugin:

# For Ubuntu/Debian
apt install docker-compose-plugin -y

# Verify installation
docker compose version

4. Create Installation Directory

Create a dedicated directory for Hexactyl:

mkdir -p /etc/hexactyl
cd /etc/hexactyl

5. Download Hexactyl

Clone the Hexactyl repository:

git clone https://github.com/hexactyl/hexactyl.git .
git checkout main

6. Configure Environment

Create and configure the environment file:

cp .env.example .env

# Generate an application key
sed -i "s|APP_KEY=.*|APP_KEY=$(openssl rand -base64 32)|" .env

# Edit other settings as needed
nano .env

7. Start Hexactyl

Launch all Hexactyl services:

docker compose up -d

8. Verify Installation

Check that all containers are running:

docker compose ps

# You should see containers for:
# - hexactyl-panel
# - hexactyl-daemon
# - hexactyl-database
# - hexactyl-cache
# - hexactyl-webserver

Docker Installation

Hexactyl is designed to run in Docker containers for isolation and easy management:

Docker Configuration

The default Docker Compose file includes all necessary services. You can customize the configuration in docker-compose.yml:

  • Panel: The main web interface (port 80/443)
  • Daemon: Game server management service
  • Database: MySQL 8.0 for data storage
  • Cache: Redis for session and queue management
  • Webserver: Nginx for reverse proxy

Customizing Ports

To change default ports, edit the docker-compose.yml file:

services:
  webserver:
    ports:
      - "8080:80" # HTTP
      - "8443:443" # HTTPS
💡 Production Tip: Always use Docker volumes for persistent data. This ensures your data survives container restarts and updates.

Web Server Configuration

Configure Nginx as a reverse proxy for Hexactyl:

Basic Nginx Configuration

Create an Nginx server block for Hexactyl:

server {
  listen 80;
  server_name panel.yourdomain.com;
  return 301 https://$server_name$request_uri;
}

server {
  listen 443 ssl http2;
  server_name panel.yourdomain.com;

  ssl_certificate /etc/letsencrypt/live/panel.yourdomain.com/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/panel.yourdomain.com/privkey.pem;

  location / {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}

SSL Certificate Setup

Use Certbot to obtain free SSL certificates:

apt install certbot python3-certbot-nginx -y

# Obtain certificate
certbot --nginx -d panel.yourdomain.com

# Auto-renewal is configured automatically

Database Setup

Hexactyl uses MySQL 8.0 for data storage. The database is automatically configured via Docker:

Database Configuration

Configure database settings in your .env file:

DB_CONNECTION=mysql
DB_HOST=database
DB_PORT=3306
DB_DATABASE=hexactyl
DB_USERNAME=hexactyl
DB_PASSWORD=your_secure_password_here

Manual Database Access

To access the database directly:

# Connect to the database container
docker exec -it hexactyl-database mysql -u hexactyl -p

# Or access from host
mysql -h 127.0.0.1 -P 3306 -u hexactyl -p hexactyl
⚠️ Security Warning: Never use default or weak database passwords. Generate a strong password using: openssl rand -base64 32

Environment Configuration

Customize Hexactyl by editing the .env file:

Essential Variables

  • APP_KEY - Application encryption key (required)
  • APP_URL - Your panel's URL
  • APP_ENV - Environment (production/development)
  • APP_DEBUG - Debug mode (false in production)

Complete Configuration Example

# Application
APP_NAME=Hexactyl
APP_ENV=production
APP_KEY=base64:your_generated_key_here
APP_URL=https://panel.yourdomain.com
APP_DEBUG=false

# Database
DB_CONNECTION=mysql
DB_HOST=database
DB_PORT=3306
DB_DATABASE=hexactyl
DB_USERNAME=hexactyl
DB_PASSWORD=secure_password_here

# Cache/Queue
REDIS_HOST=cache
REDIS_PORT=6379
💡 Configuration Tips: After changing environment variables, restart the containers with: docker compose restart
