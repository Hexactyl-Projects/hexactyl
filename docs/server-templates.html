Server Templates

Create and manage server templates for quick deployment

What Are Server Templates?

Server templates are pre-configured server setups that you can save and reuse. Instead of manually configuring each new server with the same settings, you can create a template once and deploy it multiple times with just a few clicks.

Templates are perfect for:

  • Standardized Configurations: Ensure all servers of a type have identical settings
  • Quick Deployment: Spin up new servers in seconds instead of minutes
  • Reduced Errors: Eliminate manual configuration mistakes
  • Team Management: Allow team members to deploy servers without full admin knowledge
  • Game Updates: Quickly deploy servers with the latest game version

Template Components

Each template can include:

  • Game Type: Minecraft, ARK, Rust, etc.
  • Game Version: Specific version or latest
  • Startup Command: Custom startup parameters
  • Environment Variables: Game-specific settings
  • Resource Limits: CPU, RAM, and disk allocations
  • Default Files: Configuration files to include
  • Port Allocations: Network port mappings

Creating Templates

Learn how to create your own server templates:

Via Panel Interface

  1. Navigate to Admin Panel → Templates
  2. Click Create New Template
  3. Fill in the template details:
    • Template Name: Descriptive name (e.g., "Minecraft 1.20.4 Survival")
    • Description: What this template is for
    • Visibility: Public (all users) or Private (admin only)
  4. Configure the server settings
  5. Click Save Template

Creating from Existing Server

You can create a template from an existing server:

  1. Go to the server you want to use as a template
  2. Click Actions → Create Template
  3. Review the settings that will be saved
  4. Name your template and add a description
  5. Click Create Template

Template Configuration Example

# Example Minecraft Template Configuration
{
  "name": "Minecraft 1.20.4 Survival",
  "description": "Standard Minecraft survival server with popular plugins",
  "egg_id": 1,
  "docker_image": "ghcr.io/parkervcp/yolks:minecraft_java",
  "startup": "java -Xms${SERVER_MEMORY}M -Xmx${SERVER_MEMORY}M -jar server.jar nogui",
  "environment": {
    "SERVER_MEMORY": "2048",
    "MC_VERSION": "1.20.4",
    "EULA": "TRUE"
  },
  "limits": {
    "memory": 2048,
    "cpu": 100,
    "disk": 10240
  },
  "allocation": {
    "default": 25565
  }
}
💡 Template Tips: Give your templates descriptive names so you can easily identify them. Include the game version in the name for quick reference.

Using Templates

Deploy servers quickly using your saved templates:

Deploying from Template

  1. Navigate to Servers → Create New Server
  2. Select From Template option
  3. Choose your template from the list
  4. Customize settings if needed (name, node, allocation)
  5. Click Create Server

Template Selection

When creating a new server, you'll see available templates:

  • My Templates: Templates you created
  • Public Templates: Templates shared by admins
  • Game Templates: Official templates for each game type

Overriding Template Settings

You can modify template settings before deployment:

  • Server Name: Give your server a unique name
  • Node: Choose which node to deploy to
  • Resources: Adjust CPU, RAM, or disk limits
  • Environment: Modify game-specific variables
  • Startup: Customize the startup command
⚠️ Note: Overriding template settings only affects the new server. The original template remains unchanged for future deployments.

Managing Templates

Keep your templates organized and up to date:

Editing Templates

  1. Go to Admin Panel → Templates
  2. Find the template you want to edit
  3. Click Edit
  4. Make your changes
  5. Click Save Changes

Deleting Templates

Remove templates you no longer need:

  1. Navigate to the template
  2. Click Actions → Delete
  3. Confirm the deletion
⚠️ Caution: Deleting a template does not affect servers already created from it. However, you cannot use the template for new deployments.

Template Organization

Keep templates organized with these strategies:

  • Naming Convention: Use consistent naming (e.g., "Game - Type - Version")
  • Descriptions: Add detailed descriptions for team members
  • Categories: Group templates by game or purpose
  • Visibility: Set appropriate visibility levels

Template Best Practices

  • Test templates before making them public
  • Update templates when game versions change
  • Document any special configuration requirements
  • Back up important templates regularly
  • Review and clean up unused templates
💡 Pro Tip: Create a "master" template for each game type, then clone and customize it for specific server configurations. This maintains consistency while allowing flexibility.

Template API

Manage templates programmatically using the API:

List Templates

GET /api/application/templates

# Response
{
  "data": [
    {
      "id": 1,
      "name": "Minecraft Survival",
      "description": "Standard Minecraft server",
      "visibility": "public"
    }
  ]
}

Create Template

POST /api/application/templates

{
  "name": "New Template",
  "description": "Template description",
  "egg_id": 1,
  "docker_image": "ghcr.io/parkervcp/yolks:minecraft_java",
  "startup": "java -jar server.jar nogui",
  "environment": {},
  "limits": {
    "memory": 2048,
    "cpu": 100,
    "disk": 10240
  }
}

Delete Template

DELETE /api/application/templates/{id}
